Crear una base de datos local requiere primero instalar el cliente de PostgreSQL en WSL2 con el comando `sudo apt install postgresql-client -y`, el cual provee las herramientas `psql` y `pg_dump` necesarias para el resto del proceso. Luego se debe identificar la estructura actual de la base de datos en la nube de Neon, la cual está definida en los archivos de migración dentro de `backend/` (`migracion_v4.sql`, `migracion_v5_*.sql`), listándolas en el orden en que fueron creadas para poder recrear el mismo esquema en local ejecutándolas contra la base de datos `ministerio_infantil` en PostgreSQL local en WSL2. Una vez replicada la estructura, se extrae la string de conexión de Neon desde `backend/.env` (variable `DATABASE_URL`) y se usa `pg_dump` con la opción `--data-only` y `--no-owner` para exportar únicamente los datos reales desde la nube a un archivo `respaldo_neon.sql`. Ese archivo se restaura en la base de datos local usando `psql`. Para complementar, se ejecuta el seed no destructivo `backend/src/seed_produccion.ts` que garantiza datos semilla adicionales. Finalmente, se configuran los archivos `backend/.env` y `backend/.env.test` con las variables de conexión local (`DB_HOST=localhost`, `DB_PORT=5432`, `DB_NAME=ministerio_infantil`, `DB_USER=postgres`, `DB_PASSWORD=postgres`) y se verifica que tanto la conexión como los tests (`npm test`) funcionen correctamente contra la base de datos local con datos reales.
